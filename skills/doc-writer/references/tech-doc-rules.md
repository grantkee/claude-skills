# Shared constraints for all editing agents

These rules apply to EVERY editing pass. Read them before touching the target file.

1. **Never change technical meaning.** If you are unsure whether a rewrite alters meaning, leave the sentence alone.
2. **Do not touch text inside backticks, code blocks, or markdown links.** Inline code (`like this`), fenced code blocks, and link text/URLs are off-limits.
3. **Use the document's established terminology.** Do not invent synonyms for technical terms already used in the file.
4. **Preserve cross-references, anchors, and link targets.** Do not rename headings that other documents link to without updating the links.
5. **The reader is a developer.** Explain domain concepts; do not explain programming basics.
6. **Make minimal changes.** Each edit must be justifiable by a specific rule in your reference file. If a sentence is technically precise but stylistically imperfect, prefer precision.
7. **Report changes when done.** After editing, state the number of changes you made and which rules drove them.
